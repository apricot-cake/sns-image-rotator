# SNS Image Rotator（SNS画像回転）

SNS のタイムラインに流れてくる横倒しの画像を、その場で正しい向きに回転できる Chrome 拡張機能。X (Twitter) と Bluesky (bsky.app) に対応し、どちらもタイムラインと拡大表示（ライトボックス）の両方で回せる。

![横倒しのイラストを右クリックメニューから 90° 回転させ、正しい向きで大きく表示するまで](docs/screenshot.png)

UI は多言語対応（`_locales`, en / ja / ko / zh_CN / zh_TW）: 右クリックメニュー・ホバーボタンのツールチップ・ポップアップの文言と、拡張機能の表示名を、ブラウザの言語に合わせて出し分ける。表示名は "SNS" が通じる地域（日本・韓国）では「SNS画像回転」「SNS 이미지 회전」、通じない地域では英語 "Social Media Image Rotator" や中文「社交媒体图片旋转器」を使う。

## 既製の回転拡張機能が X で効かない理由

X はツイート画像を `<img>` ではなく `<div>` の CSS 背景画像として描画し、`<img>` は透明な当たり判定として重ねている。汎用の回転拡張機能は `<img>` を回すため、X 上では見た目が変わらない。本拡張は背景 `<div>` と `<img>` の両方を回すことでこれを解決する。

## 機能

- **ホバーボタン**: 画像にマウスを乗せると右上（拡大表示では右下）に左回転・右回転の 2 ボタンが出る。クリックごとに 90° 回転する。ポップアップ設定でオンオフ可（既定オフ）
- **右クリックメニュー**: 「画像を左/右に 90° 回転」。常に有効
- **対象**: タイムライン・引用ポストの写真、および拡大表示（ライトボックス）

回すだけでなく、回転後の表示サイズも作り直す。単に 90° 傾けると縦横比が変わって余白や見切れが出るため。

- **拡大表示**: 回転後の画像を元の枠ではなくモーダルの使える領域に合わせて拡大する。横倒しの投稿を起こすと、画面の高さをフルに使って大きく表示される
- **タイムラインの単独写真**: 枠自体を回転後の縦横比に作り替え、本文と同じコンテンツ列の幅いっぱいに（余白なく）表示する。縦長ファイルの横向き絵など、元は列幅より狭く表示されていた画像も回転後は列幅まで拡大される
- **縦に長くなりすぎる場合**: ビューポート（画面の見えている範囲）の高さを超えないよう抑え、画像全体が一度に見えるようにする。このとき枠は画像にぴったり合わせて（灰色の帯を出さず）列の中央に置く
- **複数枚グリッド・動画サムネイル**: 段組みやプレイヤーを崩さないよう、従来どおり元の枠内に縮小する

## インストール

[Chrome ウェブストア](https://chromewebstore.google.com/detail/affodfbfgaikjohkhnfmjalnapgjfklf) から入れられる。

Chrome の「セーフ ブラウジング保護強化機能」が拡張機能を信頼済みと判定するまでには数か月かかるため、公開直後はインストール時に警告が出る。危険と判定されたわけではなく、実績を積むまでの既定の挙動。

### 開発版を読み込む

1. `npm install && npm run build`
2. Chrome の `chrome://extensions` で「デベロッパー モード」をオン
3. 「パッケージ化されていない拡張機能を読み込む」で `.output/chrome-mv3` を選択

コード更新後は同じ画面で拡張機能の再読み込みボタン（↻）を押す。

## 開発

[WXT](https://wxt.dev/) + TypeScript。

```sh
npm install
npm run dev      # 自動リロード付き開発
npm run build    # 本番ビルド
npm run compile  # 型チェック
```
